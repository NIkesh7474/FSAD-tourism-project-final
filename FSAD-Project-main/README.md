# FSAD-Project
Project Title:
Support tourists with homestay ideas and nearby tourism places

Project Overview:
This project is a web or mobile platform designed to help travelers find comfortable homestay options and explore nearby tourist attractions. The main goal is to improve the travel experience by offering personalized recommendations, local insights, and easy booking facilities. The system connects tourists with homestay hosts and local guides while allowing an admin to manage everything.

Objectives:
The platform aims to simplify travel planning by helping users discover affordable and authentic homestays. It also helps tourists explore nearby attractions with the help of local guides who provide real insights. Another objective is to support local communities by promoting their homestays and tourism knowledge.

Main Users and Their Roles

Admin:
The admin controls the entire platform. The admin can add, update, or remove homestay listings and tourist place information. The admin manages users including tourists, hosts, and local guides. The admin monitors bookings and handles complaints or issues. The admin ensures the platform runs smoothly.

Homestay Host:
The host is a person who owns or manages a homestay. The host can create a profile and list their homestay with details like location, price, facilities, and images. The host can manage booking requests and confirm or reject them. The host can communicate with tourists and update availability.

Tourist:
The tourist is the user who wants to travel. The tourist can search for homestays based on location, budget, and preferences. The tourist can view details, check availability, and book a stay. The tourist can explore nearby tourist attractions and get suggestions. The tourist can also review and rate homestays and guides.

Local Guide:
The local guide provides information about tourist attractions. The guide can add details about places such as history, best time to visit, and travel tips. The guide can recommend hidden spots and local experiences. The guide can interact with tourists and answer their queries.

Key Features:

User Registration and Login
Users can sign up and log in as admin, host, tourist, or guide.

Homestay Listings:
Hosts can add detailed homestay listings including images, pricing, and amenities.

Search and Filters:
Tourists can search homestays using filters like location, price range, and ratings.

Booking System:
Tourists can book homestays and hosts can manage booking requests.

Nearby Attractions:
The platform shows nearby tourist places along with descriptions and recommendations.

Personalized Recommendations:
The system suggests homestays and places based on user interests and previous searches.

Reviews and Ratings:
Tourists can give feedback which helps others make better decisions.

Communication System:
Tourists and hosts or guides can chat or message each other.

Admin Dashboard:
Admin has a dashboard to monitor users, listings, and platform activity.

Technology Stack Suggestion:

Frontend:
HTML CSS JavaScript
Frameworks like React or Angular can be used for better user experience

Backend:
Java with Spring Boot or Node JS

Database:
MySQL or MongoDB

APIs:
Google Maps API for location and distance
Payment gateway API for bookings

System Flow:

The tourist registers and logs into the platform. The tourist searches for a homestay in a specific location. The system displays available homestays along with nearby attractions. The tourist selects a homestay and sends a booking request. The host receives the request and confirms it. The tourist can view local recommendations provided by guides. After the stay the tourist gives feedback.

Advantages:

Provides a complete travel solution in one place
Supports local businesses and tourism
Offers personalized and reliable recommendations
Improves user convenience and travel planning
